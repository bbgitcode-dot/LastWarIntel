# SENTINEL

> **The Philosophy of Sentinel**

Version: **v0.8.0**

---

# Why Sentinel Exists

Sentinel was never intended to become another dashboard.

It was never intended to become another statistics page.

It was never intended to become another OCR project.

Those are implementation details.

The purpose of Sentinel is much larger.

Its mission is to transform overwhelming amounts of game information into strategic knowledge that enables alliance leadership to make better decisions.

---

# The Problem

Competitive Last War servers generate enormous amounts of information.

Every day produces:

* hundreds of player changes
* alliance power fluctuations
* transfers
* whale movement
* activity changes
* structural changes
* recruitment opportunities

Human commanders cannot process everything.

Most leadership teams therefore rely on

* screenshots
* spreadsheets
* intuition
* fragmented experience

This approach does not scale.

Important information is missed.

Strategic opportunities disappear unnoticed.

---

# Sentinel's Mission

Sentinel continuously answers one simple question:

> **What deserves our attention today?**

Not:

> What happened?

Not:

> Show me another graph.

Instead:

> What should I care about?

This distinction defines the entire project.

---

# Sentinel is an Intelligence System

Sentinel transforms information into knowledge.

Every component exists for exactly one purpose:

Reducing uncertainty.

---

The system follows a strict progression.

```text
Information

↓

Observations

↓

Knowledge

↓

Understanding

↓

Action
```

Everything inside Sentinel supports this transformation.

---

# Information has no value

Raw information alone creates no advantage.

Knowing that

* an alliance lost 18%

or

* two whales transferred

does not automatically improve decision making.

Information only becomes valuable once it is interpreted.

---

Therefore Sentinel never stops at information.

It always asks:

Why?

What does this mean?

What should happen next?

---

# Explainable Intelligence

Sentinel deliberately avoids black-box decisions.

Every recommendation must be explainable.

Every recommendation must answer

Why?

Every conclusion can be traced backwards.

```text
Recommendation

↓

Value

↓

Assessment

↓

Reasoning

↓

Indicators

↓

Facts

↓

Observations
```

Nothing is hidden.

Nothing is guessed.

Everything is reproducible.

---

# Intelligence is Layered

Knowledge is created gradually.

Each layer increases abstraction.

```text
Observation

↓

Difference

↓

Fact

↓

Indicator

↓

Reasoning

↓

Assessment

↓

Value

↓

Recommendation
```

Skipping layers is forbidden.

Each layer has exactly one responsibility.

---

# Facts are Reality

Facts describe objective observations.

Examples:

Alliance lost 12% power.

Officer transferred.

Whale joined.

Facts never interpret.

Facts never speculate.

Facts never recommend.

They simply describe reality.

---

# Indicators describe Characteristics

Indicators summarize facts into strategic properties.

Examples:

Structural Health

Recruitability

Talent Value

Whale Density

Elite Density

Indicators answer

How healthy is this alliance?

instead of

What happened yesterday?

---

# Reasoning builds Understanding

Facts alone rarely explain a situation.

Reasoning combines multiple observations.

Example

```text
Officer left

+

Alliance lost power

+

Activity declined

↓

Alliance stability decreasing
```

Reasoning produces hypotheses.

Not priorities.

---

# Assessments describe Situations

Assessments answer

What is happening?

Examples:

Recruitment Window

Alliance Collapse

Whale Migration

Leadership Risk

Transfer Winner

Transfer Loser

Assessments represent strategic situations.

They intentionally contain no prioritization.

---

# Values prioritize Decisions

Different objectives require different priorities.

Recruitment does not care about exactly the same things as diplomacy.

Therefore Sentinel introduces Values.

Examples:

Recruitment Value

Threat Value

Diplomacy Value

Expansion Value

Retention Value

Values never create knowledge.

They prioritize existing knowledge.

---

# Recommendations support Humans

Recommendations are the final output.

They answer

What should we do?

Example

```text
Assessment

Recruitment Window

↓

Recruitment Value

92

↓

Recommendation

Contact alliance leadership within 48 hours.
```

The recommendation remains explainable.

Every supporting observation remains accessible.

---

# WatchTargets

A WatchTarget is not merely an alliance.

It is Sentinel's operational representation of an entity.

It acts as the digital twin of a strategic object.

A WatchTarget gradually accumulates intelligence.

Examples:

Assessment

Recruitment Value

Decision Snapshot

History

Notes

Future versions may also contain

Threat Value

Diplomacy Value

Expansion Value

Trend History

Contact History

WatchTargets evolve over time.

---

# Intelligence Modules

Every Intelligence Module answers exactly one strategic question.

Examples

Growth Intelligence

Which entities are growing?

Whale Intelligence

Which strategically relevant players moved?

Structural Health Intelligence

How healthy is this alliance?

Recruitment Intelligence

How attractive is this alliance for recruiting?

Opportunity Intelligence

Which opportunities currently exist?

Modules never solve multiple problems simultaneously.

---

# Intelligence Modules never communicate directly

This is one of Sentinel's most important design rules.

Modules communicate through shared knowledge.

Not through direct dependencies.

Shared knowledge includes

Facts

Indicators

Assessments

This keeps modules independent.

It also makes future extensions significantly easier.

---

# Deterministic Intelligence

Sentinel is deterministic.

The same observations always produce the same conclusions.

Randomness has no place in strategic intelligence.

Confidence expresses uncertainty.

Randomness does not.

---

# Scores do not make Decisions

Scores exist for exactly one reason:

Prioritization.

Nothing more.

They never replace assessments.

Example

Wrong

```text
Recruitment Score

↓

Recommendation
```

Correct

```text
Assessment

↓

Recruitment Value

↓

Recommendation
```

The assessment explains.

The value prioritizes.

---

# Human First

Sentinel assists commanders.

It never replaces them.

The final decision always belongs to humans.

Sentinel provides

Knowledge

Context

Evidence

Recommendations

Leadership provides

Experience

Diplomacy

Timing

Responsibility

---

# Long-Term Vision

Sentinel should evolve into a strategic intelligence platform capable of answering questions such as

Which alliance is likely to collapse?

Which whales are most likely to transfer?

Which recruitment opportunity deserves immediate attention?

Which server will become unstable next?

What changed overnight?

What should the president focus on today?

The objective is not automation.

The objective is better strategic decisions.

---

# One Sentence

If Sentinel could be summarized in one sentence, it would be this:

> **Sentinel transforms observations into explainable strategic intelligence that enables better human decisions.**
