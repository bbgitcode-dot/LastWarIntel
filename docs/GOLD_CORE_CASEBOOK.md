# GOLD CORE CASEBOOK

Version: v0.9.5.144

This document defines the generated Gold Core Casebook introduced by Gold Core Strike V.
The live casebook is generated by `ground_truth_validator.py` in the selected benchmark
output directory as `GOLD_CORE_CASEBOOK.md`.

## Truth hierarchy

For every Gold Core case, Sentinel uses this order:

1. `gold_core_failure_class`
2. `gold_core_failure_domain`
3. `gold_core_fix_lane`
4. `gold_core_resolution_action` / `gold_core_resolution_lane`
5. Strike V fallback heuristic only when no established classification exists

A fallback heuristic may add context but may never overwrite established validator triage.

## Stable case identity

Benchmark cases use `S<server>-R<rank>` as their stable case ID. Expected or observed
display text may change during correction; such display changes must not split Failure Memory.

## Generated outputs

- `gold_core_case_explorer.json`
- `gold_core_case_explorer.xlsx`
- `GOLD_CORE_CASEBOOK.md`
- enriched `gold_core_analytics_report.json`
- enriched `gold_core_analytics_report.xlsx`
- `gold_core_failure_memory.json` (Failure Memory 2.0)

## Safety

All Strike V outputs are read-only quality intelligence. They do not modify OCR exports,
snapshots, Ground Truth, DataGuard decisions, or Operational Truth.
